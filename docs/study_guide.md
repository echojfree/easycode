## 1. 项目概览 (Project Overview)

`easycode` 是一个基于 Python 实现的类 Claude Code 终端 AI 助手。它的核心目标是实现一个能够通过调用工具（Tools）来操作本地文件、执行 Bash 命令并进行多轮推理的 Agent 循环。

## 2. 核心逻辑深度解析

要掌握这个项目，必须理解以下三个关键组件的工作原理：

### A. Agent 循环 (The Query Loop)
这是项目的“大脑”。在 `scc/agent.py` 的 `QueryEngine.submit_message` 方法中，实现了一个 `while` 循环：
1. **构造 Prompt**: 将历史消息 + 系统指令 + 当前工具 Schema 发送给 LLM。
2. **解析响应**: 检查 LLM 返回的是“普通文本”还是“工具调用 (`tool_calls`)”。
3. **执行工具**: 如果是工具调用，调用 `_execute_tool`，并将执行结果（成功或报错）包装成 `tool_result` 消息。
4. **反馈循环**: 将结果塞回 `messages` 历史中，再次请求 LLM，直到 LLM 给出最终回答或达到 `MAX_ITERATIONS` 上限。

### B. 工具系统 (Tool System)
每个工具（如 `edit_file_tool.py`）都遵循统一的接口：
- `input_schema`: 定义 LLM 调用时必须提供的参数（JSON Schema）。
- `validate_input()`: 校验参数合法性。
- `call()`: 真正的业务逻辑（如读写文件）。

### C. 状态管理 (Context & Messages)
`self.messages` 存储了整个对话的“记忆”。项目通过 `make_assistant_msg` 等辅助函数确保消息格式符合 Ollama/OpenAI 的 API 规范。

## 3. 如何调试 (How to Debug)

### 方法一：日志与 Print 调试 (最快)
由于这是一个交互式 CLI 程序，最直接的方法是在关键路径插入 `print` 或使用 `logging` 模块。
- **调试 Agent 决策**: 在 `scc/agent.py` 的 `_execute_tool` 处打印 `input`。
- **调试 LLM 返回**: 在 `scc/agent.py` 处理 `tool_calls` 之前，打印 LLM 的原始 JSON 响应。
- **调试工具逻辑**: 在具体的工具文件（如 `scc/tools/file_edit_tool.py`）中打印变量。

### 方法二：使用 VS Code / PyCharm 断点调试 (推荐)
1. **设置断点**:
   - 在 `scc/agent.py` 的 `submit_message` 入口处打断点，观察循环是如何跳转的。
   - 在 `scc/api.py` 的请求处打断点，查看发送给 Ollama 的完整 Payload。
2. **配置 Launch.json**:
   创建一个调试配置，将入口指向 `main.py`。
3. **观察变量**:
   在调试时重点观察 `self.messages` 的变化。你会看到消息是如何从 `user` -> `assistant (tool_call)` -> `tool (result)` -> `assistant (final text)` 演进的。

### 方法三：编写单元测试 (长效)
项目中已有 `test/` 目录。
- 运行 `python test_full.py` 可以进行集成测试。
- 针对单个工具编写测试：例如，编写一个脚本模拟 `edit_file` 的输入，验证其是否能正确修改文件。

## 5. 常见问题排查 (Troubleshooting)

- **Ollama 连接失败**: 检查 `.env` 中的 `OLLAMA_HOST` 是否正确，确保 Ollama 服务已启动。
- **模型不调用工具**: 
  - 检查 `build_system_prompt` 是否清晰地告知了工具的存在。
  - 检查 `get_api_schemas` 生成的 JSON 是否符合模型预期的格式。
  - 尝试更换更强大的模型（如 `llama3` 或 `qwen2`），小模型可能无法正确遵循 Tool Use 指令。
- **文件权限错误**: 确保运行程序的用户对当前工作目录有读写权限。
